01 — EXECUTIVE SUMMARY & VISION

The Problem Nobody Has Actually Solved
Every recruiter and HR consultant in India has lived this story: spend ₹25–75 lakh per year on Naukri Resdex or LinkedIn Recruiter, get 300 profiles, spend 3 days reviewing them manually, find that 70% are outdated or irrelevant, send 40 emails, get 8 replies, schedule 5 interviews, make 1 hire. Then do it all over again next week — with no memory of what worked, no pipeline visibility for the client, and no automation anywhere in the chain. The tools that were supposed to solve this have simply digitised the old process without rethinking it.

The Three Root Failures of Today's Recruitment Industry
•	STALE DATA: Naukri's average profile is 8 months old. LinkedIn has no freshness signal. Candidates who got hired, moved cities, or changed careers are still served as active leads.
•	FAKE AI: Every platform claims 'AI matching'. It is Boolean keyword search renamed. A Java developer with 5 years experience and a Java fresher score identically if keywords match — no semantic reasoning exists.
•	BROKEN WORKFLOW: The actual recruitment process (JD → sourcing → screening → outreach → scheduling → interview → offer → join) spans 6+ disconnected tools. Nothing talks to anything else.

TalentRadar AI — The Complete Answer
TalentRadar AI is not another job board, not another ATS, and not another LinkedIn scraper. It is the world's first open-source, AI-native Recruitment Operating System — a single platform that owns the entire talent acquisition lifecycle from the moment a JD is created to the moment a candidate walks through the office door on Day 1.

What TalentRadar Does Differently
•	Real-Time Candidate Intelligence: Profiles refreshed every 6–24 hours across 18 platforms. Freshness score on every profile. Dead leads automatically archived.
•	True Semantic AI: Fine-tuned LLM (not keyword search) understands career context, skill adjacency, trajectory, and implicit requirements. A 'Product Manager with fintech background' query returns fintech PMs — not every PM ever listed.
•	Complete Recruitment OS: JD creation → AI sourcing → smart shortlist → automated outreach → interview scheduling → offer tracking → Day 1 onboarding — one system, zero tool-switching.
•	Open Source & Permanently Free: MIT-licensed, self-hostable on a single laptop or free cloud tier. No trial. No credit card. No feature gates. Full platform, forever.
•	2026 Stack: Built on the cutting edge — Bun.js runtime, Hono framework, pgvector in Postgres, Ollama for local LLMs, tRPC for type-safe APIs, Turso for edge SQLite, Temporal.io for durable workflows.
•	Explainable AI: Every match comes with a plain-English explanation. No black box. Recruiters see exactly why a candidate scored 87/100.
•	Candidate-First Design: Candidates get a portal too — they see their application status in real time, can update their profile, and receive personalised career suggestions from the AI.

The Vision
By 2027, TalentRadar AI will be the operating system that every Indian recruitment consultant, HR team, and hiring manager runs their practice on — the way Salesforce became the OS for sales. We will get there not by charging ₹75 lakh/year like Naukri, but by being permanently free for individuals and small teams, and charging only at enterprise scale where we deliver 10× the value at 20% the cost.

03 — PRODUCT ARCHITECTURE

System Architecture Overview
TalentRadar AI is designed as a modular, event-driven microservices platform with a monorepo structure. Every module is independently deployable, independently scalable, and independently testable. The architecture is intentionally over-engineered for a startup — because we expect to scale to 10M candidate profiles and 50,000 active recruiters within 24 months.

Top-Level Architecture — 8 Core Systems
System	Responsibility	Tech	Scale Target
Aggregation Engine	Scrape, parse, deduplicate candidate profiles from 18+ sources	Playwright, Scrapy, Apify, Temporal.io	50M profiles, 18 sources
AI Match Engine	Semantic JD parsing, vector search, TalentScore generation, explanations	LLaMA 3.1 / Qwen2.5, pgvector, Qdrant	10M matches/day
Recruitment OS Core	JD management, pipeline, stages, tasks, SLA tracking	FastAPI, PostgreSQL, Temporal.io	500K active pipelines
Communication Hub	Email, WhatsApp, SMS, LinkedIn sequences + tracking	Resend, Twilio, Bull queues	5M messages/month
Analytics Engine	Real-time funnel, SLA, recruiter performance, AI insights	ClickHouse, Apache Superset, tRPC	1B events/month
Knowledge Base	Skill taxonomy, salary benchmarks, market intelligence, RAG	ChromaDB, LangChain, Wikipedia crawls	500K entities
Integration Gateway	HRMS, ATS, calendar, Slack, Zapier connectors	REST, GraphQL, Webhooks, n8n	200+ integrations
Candidate Portal	Candidate-facing profile, applications, AI coaching, status	Next.js 15, tRPC, Edge Runtime	10M candidates

Data Flow Architecture
1.	Recruiter creates Job Requisition with JD text or template → JD Parser (LLM) extracts structured requirements in <2 seconds
2.	Aggregation Engine triggers targeted scraping across configured platforms → raw profiles ingested into Kafka stream
3.	Profile Processor normalises, deduplicates, enriches profiles → writes to PostgreSQL + generates vector embeddings in pgvector
4.	Match Engine performs cosine similarity + reranking → TalentScore (0–100) assigned with explanation tokens
5.	Recruiter sees ranked shortlist with AI reasoning → takes actions (email, WhatsApp, schedule, reject, save)
6.	All actions trigger state machine in Temporal.io → durable workflow moves candidate through pipeline stages
7.	Every outcome (hired/rejected/ghosted) feeds back into ML training pipeline → model improves weekly
8.	ClickHouse receives all events in real time → analytics dashboard updates in <500ms

Architecture Principles
•	Event-First: Every action in the system emits an event. Nothing is lost. Full audit trail. Time-travel debugging.
•	Offline-First AI: Ollama enables running the full AI engine locally without any external API calls — works without internet.
•	Edge-Ready: Candidate portal and recruiter dashboard both run on edge runtime (Cloudflare Workers / Vercel Edge) for <50ms globally.
•	Schema-First: tRPC + Zod means every API is type-safe end-to-end. No runtime type errors. IDE autocomplete across frontend and backend.
•	Idempotent Everything: All scraping jobs, emails, and webhooks are idempotent. Retry safely. No duplicate profiles, no double emails.

06 — AI ENGINE & KNOWLEDGE BASE

The TalentScore AI Engine — Deep Dive
The AI engine is TalentRadar's core moat. It is not a wrapper around a generic LLM with a clever prompt. It is a purpose-built, multi-stage reasoning system trained on recruitment-specific data that thinks about candidates the way a seasoned recruitment consultant does — understanding career trajectories, not just keyword lists.

TalentScore Architecture — 7 Layers
Layer	What It Does	Technology	Output

Data Quality Pipeline
9.	Ingestion: Raw scraped data → Kafka topic → consumer workers → normalisation
10.	Deduplication: Cross-source entity matching (name + email + phone + work history hash) → merge into unified profile
11.	Enrichment: Email finder (Hunter.io free tier / scraping) → phone normalisation → location geocoding (Nominatim, free)
12.	Freshness Scoring: Days since last activity on each platform → composite freshness score → stale profiles archived
13.	Quality Scoring: Completeness score → profiles missing critical fields flagged → low-quality profiles ranked lower
14.	Embedding Generation: nomic-embed-text runs on enriched profile text → 768-dim vector stored in pgvector
15.	Knowledge Graph Update: New skills, titles, companies extracted → taxonomy graph updated → affects all future matches

Anti-Detection & Ethical Scraping Principles
•	Respect robots.txt: Never scrape pages explicitly disallowed. Always check before adding a new source.
•	Rate Limiting: All scrapers configured with human-like delays (5–15 seconds between requests). Never hammer a server.
•	Proxy Rotation: Residential proxies for sensitive sources. Datacenter proxies for permissive sources.
•	User-Agent Rotation: Rotate between 50+ real browser user-agent strings. Never use default Playwright UA.
•	Official APIs First: Always prefer official APIs over scraping. Scraping is the fallback, never the first choice.
•	Data Minimisation: Only collect fields relevant to recruitment. Never store private messages or connection graphs.
•	Right to Erasure: Any candidate can request deletion of their profile from TalentRadar — processes in <24h.
