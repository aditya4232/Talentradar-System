# 1️⃣ Use Google X-Ray Search (Most Powerful & Free)

Instead of scraping platforms directly, query **Google indexed pages**.

Example search queries:

```
site:linkedin.com/in "python developer" "Hyderabad" "open to work"
site:github.com "Hyderabad" "machine learning engineer"
site:angel.co "Hyderabad" developer
site:stackoverflow.com/users "Hyderabad" python
```

This finds candidate profiles indexed by **Google**.

You can automate using **Google Search API alternatives**.

Free tools:

* SerpAPI (free tier)
* Serper.dev (cheap/free credits)
* Google Programmable Search Engine

Your bot will:

1. Input job description
2. Extract **skills**
3. Generate **Google queries**
4. Collect candidate profile links

# 2️⃣ Extract Email from Public Sources

Many developers expose emails publicly on:

* GitHub
* Stack Overflow
* Personal portfolios
* Research pages

You can parse pages using:

* Python
* BeautifulSoup
* Regex email finder

Example:

```python
import re

emails = re.findall(r"[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}", page_text)
```

# 3️⃣ Use GitHub Developer Search API

Developers often list:

* email
* location
* skills
* repos

Example query:

```
location:Hyderabad language:Python
```

API:

```
https://api.github.com/search/users?q=location:Hyderabad+python
```

Fields you get:

* username
* profile link
* repos
* contributions

From repo commits you can sometimes extract **emails**.

# 6️⃣ Auto-Match Candidate With Job Description

Use embeddings.

Libraries:

* SentenceTransformers
* FAISS

Process:

```
Job description → embedding
Candidate profile → embedding
Cosine similarity
```

Score candidate.

Example output:

```
Name: Rahul Sharma
Skills: Python, AWS, ML
Match Score: 87%
Location: Hyderabad
Profile: github.com/rahul
```
